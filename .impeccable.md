## Design Context

### Users
Three distinct user types: **CCRO** (power user and system owner — uses Meridian as their operating environment, needs speed and confidence); **OWNER** (functional owners with accountability for specific risks/controls — not compliance specialists, needs simplicity to drive engagement); **CEO** (executive scan mode — needs a 5-second "is anything on fire?" answer with one-click drill-down). Most users work on desktop internal tooling in a regulated UK fintech context.

### Brand Personality
**Professional. Modern. Valuable.**

The interface should feel like premium internal tooling — authoritative without being bureaucratic, modern without being gimmicky, valuable in the sense that every screen earns its place. Think Bloomberg terminal meets Notion — information-dense but never overwhelming, opinionated but never in the way.

### Aesthetic Direction
- **Primary mode**: Light mode as the presented default; dark mode as a supported toggle
- **Palette**: Deep indigo-to-purple brand (`#311B92` → `#673AB7` → `#7B1FA2`) with warm off-white surfaces (`#F8F7F4`). Accent colours — emerald for health/pass, amber for warning, red for critical
- **Typography**: DM Sans (body) + Plus Jakarta Sans (headings) — a deliberate, considered pairing
- **Motion**: Purposeful only — entrances, state changes, status transitions. No decorative loops
- **Anti-patterns to avoid**: Generic AI slop aesthetic; walls of text; static bento cards with no interactivity; glassmorphism used decoratively; identical card grids; bounce/elastic easing; gradient text for metrics

### What It Should NOT Look Like
- Generic SaaS dashboards (every Vercel/Linear clone)
- Stale, static compliance spreadsheet replacements
- Walls of text or read-only information dumps
- Inconsistent panel/modal sizes across different sections

### Design Principles
1. **Direct attention, don't list information** — every screen should answer a question, not present a catalogue
2. **Consistency is trust** — panels, cards, and detail views should share a visual language; inconsistency signals fragility
3. **Innovation through interaction, not decoration** — interest comes from things responding to the user, not from visual noise
4. **Status is always visible** — RAG states, overdue counts, and health scores should be impossible to miss
5. **The right amount of density** — information-rich but never cluttered; if it doesn't help the user act, it shouldn't be there
